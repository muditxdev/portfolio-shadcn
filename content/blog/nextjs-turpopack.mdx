## What is Turbopack?

Turbopack is the next-generation bundler from the creators of Webpack, designed to provide **faster cold starts, improved incremental builds, and reduced build times**. It leverages Rust for optimized performance and offers a much-needed alternative to traditional JavaScript-based bundlers.

## Getting Started with Next.js 15 & Turbopack

To enable Turbopack in your Next.js project, update your `next.config.js`:

```javascript
module.exports = {
  experimental: {
    turbo: true,
  },
};
```

Then, start your Next.js development server using:

```sh
pnpm next dev --turbo
```

### 2. Implement Efficient Code Splitting

## Common Issues & Fixes

### **1. Turbopack Not Enabling?**
Ensure you're on Next.js 15+ and have `experimental.turbo` enabled in `next.config.js`.

### **2. Slow Build Times?**
Try removing unnecessary modules from `node_modules` and clearing your cache:
```sh
rm -rf .next/cache && pnpm next dev
```

### **3. HMR Not Working?**
If Hot Module Replacement isn't working as expected, restart the dev server:
```sh
pnpm next dev --turbo
```
